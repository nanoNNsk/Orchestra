# Orchestra

> Chat with an AI company, not a chatbot.

Orchestra is a chat interface where a team of specialized AI agents works
together like a real company. You talk to one person — the **CEO** — and the
rest of the organization handles the work behind the scenes.

You are the **Chairman**. You set the direction; the company executes.

## How it works

```
  Chairman (you)  ⇄  CEO  ──▶  delegates to the right specialists
                      ▲
                      └──────  collects results, summarizes, reports back
```

- **You talk to the CEO only.** Tell the CEO what you want. They take it from there.
- **The CEO delegates and reports.** Work is routed to the relevant specialists;
  you get one consolidated summary back instead of noise from every agent.
- **Agents are persistent roles, not disposable tasks.** Each agent holds a
  position with its own skill set and stays on the team waiting for work — they
  are not spawned for a single task and discarded afterward.

## The team

Each agent is a domain expert occupying a standing role. The roster is
configurable — add or remove positions to fit the work. For example:

| Role | Focus |
| --- | --- |
| **CEO** | Receives direction, plans, delegates, reports back |
| Tech Lead | Technical direction and review |
| Developer | Implementation |
| Graphic | Visual design and assets |
| UX/UI | Product and interaction design |
| Manga Writer | Story and script |
| Illustrator | Artwork |
| Fortune Teller | Readings and forecasts |
| Economist | Market and economic analysis |
| Finance Expert | Financial planning and advice |

## Why Orchestra

A single agent that spawns helpers and tears them down loses context between
every task. Orchestra keeps a **stable org chart**: persistent roles accumulate
context, the CEO owns the conversation with you, and you stay focused on
*what you want* — not on micromanaging *who does it*.
